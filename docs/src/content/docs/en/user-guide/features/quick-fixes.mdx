## What It Is

Automatic refactoring handles repetitive, error-prone edits such as port connections, parameter assignments, procedural blocks, control flow, and expression rewrites. In VS Code, you trigger these operations from the <Codicon name="lightbulb" label="Lightbulb menu icon" color="#facc15" /> lightbulb menu, also called Code Actions.

## Supported Actions

Place the cursor near an instance, port list, procedural block, expression, integer literal, or diagnostic, then open the lightbulb menu. Vide currently supports:

### Instances, Ports, and Parameters

- `Fill connections`
- `Fill parameters`
- `Convert ordered port connections to named connections`
- `Convert ordered parameter assignments to named assignments`
- `Remove empty port connections`
- `Add explicit empty port connection`
- `Add empty instance port list`
- `Expand named port shorthand` / `Collapse named port to shorthand`
- `Convert ANSI port declarations to non-ANSI` / `Convert non-ANSI port declarations to ANSI`
- `Split declaration`, `Sort named parameter assignments`, and `Sort named port connections`

### Procedural Blocks and Control Flow

- `Convert to always_comb`, `Convert to always_ff`, `Convert to always @(*)`, and `Convert to always @(...)`
- `Add default case item`
- `Invert if/else` and `Merge nested if`
- `Wrap statement in begin/end` / `Unwrap single-statement begin/end`
- `Pull assignment up` / `Pull assignment down`

### Expressions and Number Literals

- `Extract into variable`
- `Apply De Morgan's law` / `Factor De Morgan's law`
- `Remove redundant parentheses`
- `Expand compound assignment` / `Collapse compound assignment`
- Increment, decrement, compound-assignment, and expanded-assignment forms can be converted into each other, for example between `count++`, `++count`, `count += 1`, and `count = count + 1`.
- `Convert literal to binary/octal/decimal/hexadecimal` and `Remove digit separators`

### Diagnostic Quick Fixes

- `Insert missing '{token}'` when a parse diagnostic identifies a missing bracket, semicolon, or similar syntax token.

## Project Configuration

### Without Complete Configuration

Some refactorings for instances and integer literals are available in the current file. If the target module is in another file, port and parameter actions only appear after Vide scans and resolves that target module.

### With Complete Configuration

After `vide.toml` is configured, Vide finds target modules through `sources`, `include_dirs`, `defines`, and `libraries`. Filling ports, filling parameters, and converting connection styles use the actual project structure.

Quick fixes that depend on diagnostics only appear after the matching diagnostic is already visible in the current file.
